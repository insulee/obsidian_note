### 프로젝트 만들기

- 프로젝트 > 프로젝트 만들기 에서 프로젝트 템플릿을 이용해 프로젝트 생성
- 개발관련 프로젝트는 주로 **칸반** 또는 **스크럼**으로 관리
- 칸반이 적합한 상황
    - 팀의 작업이 지속적으로 발생하고, 고정된 기간이 필요하지 않을 때.
    - 작업의 우선순위가 자주 변경되며, 새로운 작업이 언제든 추가될 수 있을 때.
    - 변화가 많은 작업 환경에서 유연하게 관리가 필요할 때.
- 스크럼이 적합한 상황
    - 팀이 고정된 시간 안에 명확한 목표를 달성해야 할 때.
    - 작업을 반복적인 주기로 관리하고 싶을 때.
    - 팀 내에서 정해진 역할이 필요하고, 짧은 주기로 빠른 성과를 내야 하는 경우.
- 프로젝트 템플릿이므로 둘을 혼합해서 쓰는 것도 가능
- [[직무별 프로젝트 템플릿]]


### JIRA에서 사용하는 개념

- **에픽 (Epic)**
	- 프로젝트의 큰 기능이나 목표
	- 여러 스토리와 작업을 포함
	- 예: 사용자 관리 기능 개발
- **스토리 (Story)**
	- 스토리는 에픽을 구성하는 작은 요구사항이나 기능으로 주로 사용자 입장에서 적시
	- 에픽은 큰 목표를 설정하는 반면, 스토리는 그 목표를 달성하기 위한 구체적인 기능을 설명
	- 링크된 이슈로 작업과 연계
	- 예: 사용자가 로그인 할 수 있어야 한다. 
- **작업 (Task) / 하위작업 (Sub-Task)** [^1]
	- 스토리를 완료하기 위한 구체적인 실행 단위
	- 작업은 개발자가 실제로 해야 할 세부적인 일
	- 작업은 기술적인 세부사항이나 해야 할 액션
	- 예: 로그인 화면 UI 개발, 비밀번호 암호화 처리, 로그인 실패 시 에러 메시지 표시
- **스프린트 (Sprint)**
	- 일정한 기간 동안 완료할 작업을 계획하고 실행하는 단위
	- 특정 기능이나 목표를 기준으로 나뉘는 것이 아니라, 일정 기간 동안 처리 가능한 작업을 중심으로 구성하여 집중해서 목표 달성
	- 에픽 안에 여러 스프린트가 있을 수 있고, 하나의 스프린트에 여러 에픽의 작업을 포함할 수도 있음
		- 에픽이 큰 작업일 경우 여러 스프린트가 있을 수 있음
		- 우선순위에 따라 여러 에픽의 작업 또는 팀원별 작업을 동시에 배정  
	- 예: 로그인UI 및 입력 폼 구현
- **구체적인 예시**
	- 에픽: 사용자 관리 기능 개발
	- 스토리 1: 사용자 로그인 기능 구현
	    - 작업 1: 로그인 UI 디자인
	    - 작업 2: 로그인 인증 로직 개발
	    - 작업 3: 비밀번호 암호화 처리
	    - 작업 4: 로그인 실패시 에러 메시지 표시
	- 스토리 2: 사용자 등록 기능 구현
	    - 작업 1: 사용자 입력 폼 구현
	    - 작업 2: 데이터베이스에 사용자 정보 저장 로직 개발
	- 스프린트: 로그인UI 및 입력 폼 구현(3주)
		- 목표: 로그인UI 디자인 구현 완료, 사용자 입력 폼 구현 완료
- 백로그
	- 프로젝트 수행 계획을 수립하기 위한 작업의 초기 상태
	- 모든 작업 목록을 관리하는 공간
	- 에픽, 스토리, 작업을 추가하여 우선순위를 설정

### 설정 및 적용 사례

1. 프로젝트 > `프로젝트 만들기` 에서 프로젝트 템플릿을 이용해 프로젝트 생성
2. 메뉴바 > `프로젝트 설정`: 다양한 설정 가능
	- `이슈 유형` > 에픽, 작업, 스토리, 하위작업 추가 / Task 삭제 
		- 각 이슈 유형의 필드 설정 가능
		- 필요없는 필드(레이블, 팀 등)는 제거하고 필요한 필드(기한, URL 등) 추가
	- `기능` > 백로그, 캘린더, 목표, 스프린트 등 추가
3. 메뉴바 > `백로그` > `에픽` > `에픽 패널표시` > 에픽 추가
4. 백로그에 스토리, 작업을 생각나는대로 나열 후 에픽 지정 & 스토리-작업 링크설정
	- 에픽 1: 스토어 개설 및 기본 설정
		- 작업 1: 네이버스토어 등록
			- 하위 작업 1: 네이버 ID 생성
			- 하위 작업 2: 네이버 스토어 등록
			- 하위 작업 3: 통신판매업 신고증 신청
			- 하위 작업 4: 등록면허세 납부
		- 작업 2: 네이버 비즈니스 스쿨
	- 에픽 2: 제품 상세페이지 제작
		- 작업 1: 제품사진 촬영 및 편집
			- 하위 작업 1: 제품 사진 촬영
			- 하위 작업 2: 이미지 보정 및 최적화
		- 작업 2: 제품 설명 문구 작성
	- 또는, `목록` > `+하위이슈 만들기`로 스토리/작업 등록
	- 각 스토리/작업 > `하위 이슈` > 하위 작업 추가
		- 예: 네이버 ID 생성
			- 네이버 계정: id - dabitsol // pw - Dabit2439@
			- 스마트 스토어: id - dabit@dabitsol.com // pw - Dabit2439@
5. 메뉴바 > `타임라인` > 
	1. 에픽의 일정 파악
	2. 연동상황 지정
	3. 진행상황 체크 (완료, 진행중 %로)
6. 메뉴바 > `캘린더` > 에픽, 스토리, 작업의 일정 파악
	- 메뉴바 > `프로젝트 설정` > `이슈 유형`
	- start day, 기한 필드 추가
7. 메뉴바 > `백로그` > 스토리/작업을 보드로 이동해 진행상황 지정\
	- 작업, 스토리, 하위작업 완료
	- 타임라인에서 파악 가능
8. `프로젝트 설정` > `기능` > `스프린트`
9. Confluence를 이용해 작업별 위키 작성
> 통신판매업신고증 등록 (스마트스토어 오픈 이후)
> - 통신판매업 신고 신청 @정부24
> - 등록면허세 납부 @위택스/이택스
> - 통신판매업 신고증 발급/출력 @정부24
> - 통신판매업 신고번호 등록하기 @스마트스토어센터 (판매자정보 > 정보변경신청)




[^1]: 스토리는 큰 기능 단위를 설명하고, 그 기능을 달성하기 위한 구체적인 단계가 작업(Task). 
작업이 완료될 때마다 스토리가 점점 완성되어, 스토리가 완료되면 해당 기능이 구현
스토리를 작성한 후, 이를 작은 작업으로 세분화하여 팀원들이 각자 맡아서 처리할 수 있도록, 예를 들어, **스토리**는 "사용자가 비밀번호를 재설정할 수 있어야 함"이고, **작업**은 "비밀번호 재설정 로직 구현", "비밀번호 재설정 이메일 전송 구현"
주로 프로젝트/프로덕트 오너 또는 매니저가 스토리를 구상하고 그에 따른 작업을 팀원들에게 할당
팀원이 적을떄는 스토리 보다는 작업으로 할 일을 구성
