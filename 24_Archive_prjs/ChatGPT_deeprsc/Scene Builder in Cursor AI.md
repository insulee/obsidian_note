좋습니다. Cursor AI에서 FXML 파일을 사용하여 JavaFX UI를 시각적으로 편집하거나, 코드 기반으로 수정한 내용을 미리 볼 수 있는 방법, 그리고 플러그인/확장 기능 등을 포함한 가능한 모든 솔루션을 조사해드릴게요. 가능한 경우 드래그 앤 드롭 UI 편집, 코드 수정 후 미리보기(Hot Reload), 관련 플러그인까지 모두 확인하겠습니다. 조사 완료되면 자세히 정리해서 알려드릴게요.

# JavaFX FXML 편집 및 Cursor AI 활용 방안

## 1. FXML 파일의 드래그 앤 드롭 시각적 편집 방법

**JavaFX Scene Builder 활용:** JavaFX UI 레이아웃은 일반적으로 **Scene Builder** 도구를 사용해 시각적으로 편집합니다. Scene Builder는 코드 작성 없이 구성 요소를 드래그 앤 드롭하여 FXML 레이아웃을 설계할 수 있는 공식 GUI 빌더입니다 ([JavaFX Scene Builder Information - Oracle](https://www.oracle.com/java/technologies/javase/javafxscenebuilder-info.html#:~:text=JavaFX%20Scene%20Builder%20Information%20,drag%20and%20drop%20UI)). Cursor AI 자체에는 Scene Builder와 같은 시각적 편집기가 내장되어 있지 않으므로, **외부 Scene Builder와의 연동을 통해** FXML을 편집해야 합니다.

**VS Code 확장 기능 연동:** Cursor AI는 VS Code를 기반으로 한 IDE이므로 VS Code용 확장 기능을 설치하여 Scene Builder를 연동할 수 있습니다 ([Cursor – Java](https://docs.cursor.com/guides/languages/java#:~:text=Cursor%20supports%20VS%20Code%20extensions,Install%20the%20following%20manually)). 예를 들어 **“SceneBuilder extension for VS Code”**(혹은 **FXML Viewer** 확장) 설치 후 설정을 하면, Cursor 내에서 FXML 파일을 우클릭하여 **“Open in Scene Builder”** 명령으로 곧바로 Scene Builder에서 해당 FXML을 열 수 있습니다 ( [SceneBuilder extension for Visual Studio Code - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=bilalekrem.scenebuilderextension#:~:text=,click%20%27Open%20in%20Scene%20Builder) ). 이 확장 기능을 사용하기 위한 단계는 다음과 같습니다:

1. **Scene Builder 설치:** 우선 Gluon에서 제공하는 최신 JavaFX Scene Builder를 다운로드하여 설치합니다 (Windows, macOS, Linux 모두 지원) ([SceneBuilder英語版を導入しVSCodeでFXMLをSceneBuilderで開く #環境構築 - Qiita](https://qiita.com/kazushi47/items/a4dd0153426be88e7567#:~:text=%E6%9C%80%E5%BE%8C%E3%81%AB%E5%AE%9F%E9%9A%9B%E3%81%AB%E9%96%8B%E3%81%91%E3%82%8B%E3%81%8B%E8%A9%A6%E3%81%97%E3%81%A6%E3%81%BF%E3%81%BE%E3%81%97%E3%82%87%E3%81%86%E3%80%82)).
2. **확장 기능 설치:** Cursor AI의 Extensions 패널에서 “FXML” 또는 “Scene Builder”로 검색해 **SceneBuilder Extension** (예: `bilalekrem.scenebuilderextension`)을 설치합니다 ( [SceneBuilder extension for Visual Studio Code - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=bilalekrem.scenebuilderextension#:~:text=) ) ( [SceneBuilder extension for Visual Studio Code - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=bilalekrem.scenebuilderextension#:~:text=,click%20%27Open%20in%20Scene%20Builder) ). 이 확장은 VS Code/Cursor에서 FXML 파일을 열 때 외부 Scene Builder 앱을 호출해주는 기능입니다.
3. **경로 설정:** 확장 설치 후 **설정(Extensions Settings)**에서 Scene Builder 실행 파일 경로를 지정합니다. 예를 들어 Windows의 경우 `C:\Program Files\SceneBuilder\SceneBuilder.exe` 경로를 입력합니다 ([SceneBuilder英語版を導入しVSCodeでFXMLをSceneBuilderで開く #環境構築 - Qiita](https://qiita.com/kazushi47/items/a4dd0153426be88e7567#:~:text=%E3%81%99%E3%82%8B%E3%81%A8%E6%AC%A1%E3%81%AE%E7%94%BB%E9%9D%A2%E3%81%AB%E3%81%AA%E3%82%8A%E3%81%BE%E3%81%99%E3%80%82%20Image%3A%20image)) ([SceneBuilder英語版を導入しVSCodeでFXMLをSceneBuilderで開く #環境構築 - Qiita](https://qiita.com/kazushi47/items/a4dd0153426be88e7567#:~:text=%E5%AF%BE%E8%B1%A1%E3%81%AEFXML%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%92%20%E5%8F%B3%E3%82%AF%E3%83%AA%E3%83%83%E3%82%AF%20%E3%81%97%E3%80%81,png)).
4. **FXML 열기:** Cursor AI 파일 탐색기 또는 에디터에서 FXML 파일을 **오른쪽 클릭**하고 **“Open in Scene Builder”**를 선택합니다. 또는 명령 팔레트(`Ctrl+Shift+P`)에서 해당 명령을 실행할 수도 있습니다 ( [SceneBuilder extension for Visual Studio Code - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=bilalekrem.scenebuilderextension#:~:text=,click%20%27Open%20in%20Scene%20Builder) ). 그러면 설치된 Scene Builder 프로그램이 열리면서 해당 FXML 레이아웃을 시각적으로 편집할 수 있습니다. Scene Builder에서 Label, Button 등의 컨트롤을 마우스로 배치하고 속성을 수정하면 FXML 코드에 즉시 반영됩니다.

([Configure JavaFX Scene Builder | IntelliJ IDEA Documentation](https://www.jetbrains.com/help/idea/opening-fxml-files-in-javafx-scene-builder.html)) IntelliJ IDEA의 통합 Scene Builder 탭 예시: FXML 파일을 열어 시각적으로 편집하고 있는 장면. Scene Builder를 활용하면 버튼, 레이아웃 등을 드래그 앤 드롭으로 배치하고 속성을 조절할 수 있으며, 이러한 방식으로 Cursor AI에서도 외부 Scene Builder를 연동해 UI 디자인 작업을 수행한다 ([Configure JavaFX Scene Builder | IntelliJ IDEA Documentation](https://www.jetbrains.com/help/idea/opening-fxml-files-in-javafx-scene-builder.html#:~:text=IntelliJ%C2%A0IDEA%20allows%20you%20to%20open,visually%20design%20JavaFX%20interfaces%20directly)).

> **참고:** Cursor AI에서 바로 드래그 앤 드롭 편집을 하는 또 다른 방법은 없으며, **반드시 Scene Builder와 같은 외부 도구를 사용**해야 합니다. Scene Builder를 독립 실행하여 FXML 파일을 열고 편집한 뒤 저장하면, Cursor AI 편집기에 즉시 해당 변경사항이 반영됩니다. (Cursor AI는 VS Code와 동일하게 파일 시스템의 변경을 실시간 감지하여 에디터 내용을 업데이트합니다.)

## 2. FXML 코드 수정 후 즉각적 화면 미리보기/핫 리로드

**실시간 미리보기 지원 여부:** Cursor AI는 웹 개발의 Live Server처럼 **FXML을 즉시 렌더링하여 미리보기**하는 기능이나, 수정 내용을 즉각 어플리케이션 UI에 반영하는 **Hot Reload** 기능을 기본적으로 제공하지 않습니다. JavaFX FXML은 Swing이나 웹과 달리 IDE 내 실시간 미리보기 지원이 제한적이며, 주로 Scene Builder에서 디자인 단계의 미리보기를 제공합니다. 예를 들어 IntelliJ IDEA의 경우에도 편집기 내에 Scene Builder를 임베드하여 미리보기를 구현하고 있지만 ([Configure JavaFX Scene Builder | IntelliJ IDEA Documentation](https://www.jetbrains.com/help/idea/opening-fxml-files-in-javafx-scene-builder.html#:~:text=When%20you%20open%20an%20,the%20file%20in%20Scene%20Builder)), Cursor AI(Visual Studio Code 기반)는 자체 미리보기 패널이 없으므로 **외부 툴을 통한 미리보기**가 필요합니다.

**Scene Builder를 활용한 미리보기:** FXML 레이아웃을 **Scene Builder에서 열어둔 상태로** Cursor에서 코드를 수정하면, 변경 사항을 확인하기 위해 Scene Builder 쪽에서 수동으로 새로 고침해야 합니다. Scene Builder에는 _File > Revert to Saved_ 기능이 있어, Cursor (혹은 다른 IDE)에서 FXML 파일을 저장한 후 Scene Builder에서 **Revert**를 수행하면 최신 코드가 반영됩니다 ([java - JavaFX Scene Builder Refresh from Code - Stack Overflow](https://stackoverflow.com/questions/42624037/javafx-scene-builder-refresh-from-code#:~:text=Once%20you%20saved%20the%20file,me%20it%20didn%27t%20update%20automatically)). 이 과정을 통해 **코드 수정 → 저장 → Scene Builder 새로고침**으로 변경된 UI를 확인할 수 있습니다. 다소 수동적이지만, Cursor AI 환경에서 즉각적인 UI 미리보기를 확인하려면 **Scene Builder와 병행하여 사용**하는 방법이 가장 현실적인 해결책입니다.

**애플리케이션 실행을 통한 확인:** FXML은 결국 자바 어플리케이션의 UI 뷰이므로, 코드를 수정한 뒤 **프로그램을 재실행하여 화면을 확인**하는 것이 일반적입니다. Cursor AI는 내부적으로 VS Code와 유사한 실행 환경을 제공하여, 메인 클래스를 실행하거나 Maven/Gradle로 빌드 및 실행이 가능합니다 ([Cursor – Java](https://docs.cursor.com/guides/languages/java#:~:text=Running%20and%20Debugging)) ([Cursor – Java](https://docs.cursor.com/guides/languages/java#:~:text=Terminal)). FXML 수정 후 **Cursor AI에서 바로 애플리케이션을 실행**하여 창을 띄워보는 방법도 미리보기의 한 방식입니다. 다만 이 경우 매번 재실행이 필요하며, 효율적인 핫리로드를 위해서는 추가 도구가 요구됩니다.

**Hot Reload 기법:** Java 언어 자체는 기본적으로 UI 변경의 핫리로드를 지원하지 않지만, 개발 속도를 높이기 위해 **DCEVM**(Dynamic Code Evolution VM)이나 **HotSwapAgent**와 같은 도구를 사용할 수 있습니다. 이러한 도구를 사용하면 실행 중인 JavaFX 애플리케이션에서 클래스나 리소스(FXML, CSS 등)를 변경하고도 애플리케이션을 재시작하지 않고 반영시킬 수 있습니다 ([Hot Reloading · Sapphire JavaFX (SFX)](https://sfxcode.github.io/sapphire-javafx/development/hot_reload.html#:~:text=DCEVM%20together%20with%20HotSwapAgent%20can,changed%20classes%20especially%20with%20scala)). 예컨대, TornadoFX나 Sapphire SFX와 같은 프레임워크에서는 FXML을 다시 불러오는 개발용 기능을 제공하기도 합니다. 그러나 이러한 방식은 **전문적인 설정**이 필요하며, 일반적인 FXML 개발 흐름에서는 Scene Builder를 활용한 시각적 확인이나 어플리케이션의 반복 실행으로 충분히 대응하는 경우가 많습니다.

> **참고:** Cursor AI 자체에 **Hot Reload 플러그인**이 존재하는 것은 아닙니다. 웹 개발과 달리 JavaFX는 개발 중 UI를 자동 갱신하는 표준 플러그인이 없으므로, Scene Builder의 미리보기 기능과 애플리케이션 실행으로 확인하는 방식이 사실상의 대안입니다. 만약 실시간 미리보기를 중시한다면, Scene Builder 창을 동시에 띄워두고 편집하는 접근을 권장합니다.

## 3. Cursor AI에서 활용 가능한 FXML 관련 플러그인/확장 기능 목록 및 설정 방법

Cursor AI는 VS Code와 마찬가지로 다양한 확장 기능을 통해 JavaFX FXML 개발 환경을 구축할 수 있습니다 ([Cursor – Java](https://docs.cursor.com/guides/languages/java#:~:text=Cursor%20supports%20VS%20Code%20extensions,Install%20the%20following%20manually)). 최신 정보 기준으로, FXML 편집과 JavaFX 프로젝트 관리를 돕는 **주요 확장 기능 및 외부 도구**는 다음과 같습니다:

- **Extension Pack for Java** (VS Code 공식): 기본적인 Java 지원 패키지입니다. Microsoft가 제공하는 이 팩에는 Java 언어 지원, 디버거, 테스트 러너, Maven/Gradle 지원 등이 포함되어 있어 Java 프로젝트 작업의 기반을 만들어줍니다 ([Cursor – Java](https://docs.cursor.com/guides/languages/java#:~:text=Cursor%20supports%20VS%20Code%20extensions,Install%20the%20following%20manually)). Cursor AI 사용 시에도 이 팩 설치가 권장됩니다.
    
- **JavaFX Support** (`shrey150.javafx-support`): JavaFX 사용 시 발생하는 VS Code의 클래스패스 경고나 인텔리센스 문제를 해결해주는 확장입니다. 예컨대 VS Code의 Java 언어 서버(Red Hat 제공)가 JavaFX API를 인식하지 못하는 이슈를 자동으로 해결하여, FXML 및 JavaFX 관련 클래스에 대한 코드 완성 및 문서 보기가 원활하도록 도와줍니다 ( [JavaFX Support - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=shrey150.javafx-support#:~:text=JavaFX%20Support%20for%20VSCode) ) ( [JavaFX Support - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=shrey150.javafx-support#:~:text=To%20solve%20this%20inconvenience%2C%20%60javafx,classpathentry%3E%60%20whenever%20necessary) ). Stack Overflow 답변에서도 JavaFX 개발을 위해 이 확장을 설치할 것을 안내하고 있습니다 ([java - How can I use javafx in visual studio code - Stack Overflow](https://stackoverflow.com/questions/69294852/how-can-i-use-javafx-in-visual-studio-code#:~:text=First%2C%20ensure%20that%20you%27re%20using,installation%20directory%20of%20JavaFX%20SDK)). 설치 후 별도 설정 없이 작동하며, _Language Support for Java_ 확장과 함께 JavaFX 관련 경로를 `.classpath`에 추가하는 등 환경을 조정합니다.
    
- **JavaFX Scene Builder 연동 확장** (FXML Viewer/SceneBuilder Extension): 앞서 설명한 **Scene Builder와의 연동을 위한 확장 기능**입니다. 대표적으로 _FXML Viewer_(`lzw-723.fxml-viewer`)나 _SceneBuilder Extension_(`bilalekrem.scenebuilderextension`)이 있으며, 둘 다 기능은 유사합니다. 해당 확장을 설치하면 Cursor AI에서 FXML 파일을 열 때 **곧바로 Scene Builder에서 편집**할 수 있습니다 ( [SceneBuilder extension for Visual Studio Code - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=bilalekrem.scenebuilderextension#:~:text=,click%20%27Open%20in%20Scene%20Builder) ). 설치 방법 및 사용법은 앞서 **1번 섹션**에서 상세히 다룬 단계와 동일합니다. 이 확장의 설정에서 Scene Builder 경로를 지정한 후 **“Open in Scene Builder”** 명령을 사용하면, Scene Builder가 실행되며 FXML을 로드합니다.
    
- **JavaFX Essentials Pack** (`sosuisha.javafx-essentials-pack`): 이는 복수의 JavaFX 관련 확장을 한 번에 설치해주는 번들 팩입니다. JavaFX 컨트롤러 지원, FXML 언어 모드, JavaFX CSS 지원 등 유용한 툴들이 포함되어 있어 개별로 찾지 않고도 일괄 설치할 수 있습니다 ( [JavaFX Essentials Pack - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=sosuisha.javafx-essentials-pack#:~:text=Extensions%20Included) ) ( [JavaFX Essentials Pack - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=sosuisha.javafx-essentials-pack#:~:text=%F0%9F%93%A6%20FXML%20Language%20Mode) ). 구체적으로 포함된 항목들은:
    
    - _FXML Language Mode_: FXML 파일에 특화된 XML 하이라이팅 및 포매터를 제공하고, **Scene Builder로 열기** 기능을 제공합니다 ( [JavaFX Essentials Pack - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=sosuisha.javafx-essentials-pack#:~:text=%F0%9F%93%A6%20FXML%20Language%20Mode) ).
    - _JavaFX Controller Support_: FXML의 `fx:id`나 컨트롤러 클래스 연결을 검사하여 잘못된 참조를 알려주고 빠르게 수정하도록 돕습니다 ( [JavaFX Essentials Pack - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=sosuisha.javafx-essentials-pack#:~:text=%F0%9F%93%A6%20JavaFX%20Controller%20Support) ). 예를 들어 FXML에 선언된 `fx:id`에 대응하는 @FXML 필드가 컨트롤러에 없으면 경고를 표시하고 생성 옵션을 제공하는 등 생산성을 향상시킵니다.
    - _JavaFX Property Support_: JavaFX의 Property getter/setter 메소드를 자동 생성하는 유틸리티 등 JavaFX에 특화된 코드 생성 도구들이 포함됩니다 ( [JavaFX Essentials Pack - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=sosuisha.javafx-essentials-pack#:~:text=%F0%9F%93%A6%20JavaFX%20Builder%20Class%20Generator) ).
    - _JavaFX CSS Support_: JavaFX용 CSS에 대한 구문 강조와 자동 완성 지원을 추가합니다 ( [JavaFX Essentials Pack - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=sosuisha.javafx-essentials-pack#:~:text=%F0%9F%93%A6%20JavaFX%20CSS%20Support) ).
- **JavaFX SDK (외부 라이브러리)**: 플러그인은 아니지만, JavaFX 프로젝트를 Cursor AI에서 실행하려면 **JavaFX SDK (OpenJFX)**를 프로젝트에 포함시켜야 합니다. Maven 또는 Gradle을 사용한다면 의존성 추가로 해결되지만, 단순 Java 프로젝트의 경우 Gluon에서 JavaFX SDK를 받아 설정해야 합니다 ([java - How can I use javafx in visual studio code - Stack Overflow](https://stackoverflow.com/questions/69294852/how-can-i-use-javafx-in-visual-studio-code#:~:text=First%2C%20ensure%20that%20you%27re%20using,installation%20directory%20of%20JavaFX%20SDK)). VS Code 기반 IDE에서는 VM 옵션에 `--module-path "JavaFX SDK 경로/lib" --add-modules=javafx.controls,javafx.fxml` 등을 추가해주는 설정이 필요합니다 ([java - How can I use javafx in visual studio code - Stack Overflow](https://stackoverflow.com/questions/69294852/how-can-i-use-javafx-in-visual-studio-code#:~:text=Add%20this%20to%20VM%20Option%2C)). Cursor AI에서도 동일하게 **launch.json**이나 **tasks.json**에서 해당 VM 옵션을 지정하거나, Maven/Gradle로 프로젝트를 구성하여 JavaFX 라이브러리를 로드해야 합니다. 이를 통해 FXML이 포함된 JavaFX 애플리케이션을 정상 실행하고 미리볼 수 있습니다.
    

> **설치 및 설정 요약:** Cursor AI의 좌측 Extensions 패널에서 위 언급된 확장들을 검색하여 설치할 수 있습니다. 설치 후 **Settings(`Ctrl+,`)**에서 JavaFX SDK 경로나 Scene Builder 경로 등 필요한 설정값을 입력합니다. 예를 들어 _SceneBuilder Extension_의 경우 _SceneBuilder Path_를 설정하고, _JavaFX Support_의 경우 별도 설정 없이 동작합니다. 이러한 플러그인 조합으로 Cursor AI에서도 IntelliJ 못지않게 JavaFX 개발을 진행할 수 있으며, AI 보조를 받아 FXML 컨트롤러 코드를 생성하거나 설명을 얻는 등의 작업이 가능합니다.

## 4. Cursor AI 미지원 기능 보완을 위한 통합 개발 워크플로 제안

Cursor AI 단독으로도 코딩은 가능하지만, **시각적 UI 편집/미리보기 측면의 한계를 보완**하기 위해 다른 도구와 병행하는 통합 흐름을 고려할 수 있습니다. 예를 들어 **IntelliJ IDEA + Scene Builder + Cursor AI**를 조합하여 각 도구의 강점을 활용하는 방법을 제안합니다:

- **IntelliJ IDEA + Scene Builder:** IntelliJ IDEA는 JavaFX 지원이 뛰어나며, FXML에 대한 코드 완성과 내비게이션, 컨트롤러-뷰 연계 검증 등을 기본 제공합니다. 특히 IntelliJ 2024 버전 이후로는 **FXML 파일을 IDE 내부의 Scene Builder 탭에서 바로 편집**할 수 있는 Scene Builder Kit을 지원하여, IDE 내에서 드래그 앤 드롭 UI 설계를 수행할 수 있습니다 ([Configure JavaFX Scene Builder | IntelliJ IDEA Documentation](https://www.jetbrains.com/help/idea/opening-fxml-files-in-javafx-scene-builder.html#:~:text=When%20you%20open%20an%20,the%20file%20in%20Scene%20Builder)). 이를 통해 FXML 레이아웃을 수정하면 즉시 결과를 볼 수 있고, 컨트롤러 코드와 연동도 편리하게 관리됩니다. (또는 IntelliJ에서 FXML 파일을 우클릭하여 **외부 Scene Builder**로 열어 편집하는 전통적인 방식도 가능합니다 ([Configure JavaFX Scene Builder | IntelliJ IDEA Documentation](https://www.jetbrains.com/help/idea/opening-fxml-files-in-javafx-scene-builder.html#:~:text=1.%20Press%20,open%20the%20Project%20tool%20window)).) Scene Builder로 UI를 완성한 뒤에는 IntelliJ에서 프로젝트를 빌드/실행하여 UI와 로직이 의도대로 작동하는지 빠르게 테스트할 수 있습니다.
    
- **Cursor AI의 활용:** Cursor AI는 AI 코딩 비서로서, **IntelliJ 대비 뛰어난 자연어 코드 생성/수정 능력**을 제공합니다. UI를 설계한 다음 **Cursor AI에서 컨트롤러 클래스를 불러와** 필요한 이벤트 처리기나 초기화 코드를 AI에게 작성시키는 식의 활용이 가능합니다. 예를 들어, “이 FXML에 맞는 컨트롤러 코드를 생성해줘”라고 프롬프트를 입력하면 FXML 파일과 연계된 자바 컨트롤러 클래스를 생성하거나 보완해줄 수 있습니다. 또한 Cursor AI의 Chat/Agent 기능을 통해 **JavaFX 특정 문제에 대한 도움**을 얻거나, 작성된 코드를 리팩토링/디버깅하는 데에도 사용할 수 있습니다. 이렇게 **Cursor AI를 보조 도구**로 사용하면, IntelliJ에서 일일이 코딩하는 것보다 빠르게 보일러플레이트 코드를 작성하고 오류를 수정하는 데 도움이 됩니다.
    
- **협업 운영:** 한 가지 워크플로 예시는 다음과 같습니다:
    
    1. **UI 디자인:** IntelliJ IDEA에서 JavaFX 프로젝트를 생성하고 FXML 파일을 Scene Builder(내장 탭 또는 외부 앱)로 열어 UI를 설계합니다. 레이아웃을 잡고 컨트롤들의 fx:id 및 속성을 설정한 뒤 저장합니다.
    2. **코드 작성:** 프로젝트 폴더를 Cursor AI에서도 열어(동일한 작업 디렉토리를 IntelliJ와 Cursor에서 공유) 방금 만든 FXML과 컨트롤러 코드를 편집합니다. Cursor AI의 추천을 받아 컨트롤러에 @FXML 필드를 추가하거나 이벤트 핸들러 메소드를 생성합니다. 필요한 로직을 AI에게 물어보거나 자동 완성으로 채워 넣을 수 있습니다. (이때 IntelliJ도 열려 있다면 파일 동기화에 유의해야 하지만, Git 등의 VCS를 활용하거나 파일 저장을 수시로 하면 큰 문제는 없습니다.)
    3. **테스트 및 디버그:** IntelliJ에서 애플리케이션을 실행하여 UI와 코드가 잘 연계되는지 확인합니다. IntelliJ의 디버거로 문제를 추적하거나, 필요한 경우 Cursor AI에 에러 메시지를 물어보고 해결책을 얻습니다.
    4. **반복:** UI 수정이 필요하면 다시 Scene Builder로 레이아웃을 조정하고(또는 Cursor/IntelliJ에서 FXML 코드를 직접 수정하고 Scene Builder로 미리보기), 다시 Cursor AI를 이용해 변경된 UI에 맞춰 코드를 업데이트합니다. 이렇게 **Scene Builder – Cursor AI – IntelliJ** 사이를 전환하며 각각의 장점을 활용합니다.
- **대안:** 만약 IntelliJ를 사용하지 않고 **Cursor AI와 VS Code 확장만으로 작업**하려는 경우에도, 기본 개념은 동일합니다. 즉, **Scene Builder**를 통해 UI를 설계하고, **Cursor AI(또는 VS Code 환경)**에서 코드를 작성/수정하며, **빌드 도구(Maven/Gradle)**를 통해 애플리케이션을 실행하는 흐름입니다. Cursor AI는 자체적으로 터미널과 실행 기능을 제공하므로, Maven 프로젝트로 세팅한 경우 `mvn javafx:run`으로 실행하거나, 또는 Main 클래스를 열어 **“Run”** 링크를 클릭하여 바로 실행할 수도 있습니다 ([Cursor – Java](https://docs.cursor.com/guides/languages/java#:~:text=Run)). 따라서 IntelliJ 없이도 개발은 가능하지만, **프로젝트 설정**(SDK 경로, 모듈 경로 등)을 수동으로 해야 하는 번거로움이 있습니다. 초기에 IntelliJ로 프로젝트를 생성한 후 Cursor로 옮겨오는 방법도 많이 쓰이며, 이 경우 설정이 어느 정도 갖춰진 상태에서 Cursor AI의 도움을 받아 개발을 이어갈 수 있습니다.
    

결론적으로, **Cursor AI 자체는 강력한 AI 코딩 도우미**이지만, **JavaFX GUI 편집/미리보기 기능은 제한**적입니다. 최신 Cursor AI 버전 기준으로는 VS Code용 확장 기능들을 통해 대부분의 JavaFX 개발 요구를 충족할 수 있으므로, Scene Builder 연동 확장, JavaFX 지원 확장 등을 설치하여 사용하는 것이 최선입니다 ([java - How can I use javafx in visual studio code - Stack Overflow](https://stackoverflow.com/questions/69294852/how-can-i-use-javafx-in-visual-studio-code#:~:text=First%2C%20ensure%20that%20you%27re%20using,installation%20directory%20of%20JavaFX%20SDK)) ( [SceneBuilder extension for Visual Studio Code - Visual Studio Marketplace](https://marketplace.visualstudio.com/items?itemName=bilalekrem.scenebuilderextension#:~:text=,click%20%27Open%20in%20Scene%20Builder) ). 만약 이러한 구성으로도 불편함이 있다면, IntelliJ IDEA와 Scene Builder를 함께 활용하는 하이브리드 방식을 고려해볼 수 있습니다. 이를 통해 **시각적 UI 디자인의 편의성**과 **AI 코딩 보조**를 모두 누릴 수 있으며, JavaFX 개발 생산성을 극대화할 수 있을 것입니다.
