#### 해시태그(HashTag, #)

1. 해시태그(HashTag, #)는 소셜 미디어에서 키워드나 주제를 표시하는 데 사용되는 중요한 도구
	- 현재 해시태그는 사용자가 특정 주제의 콘텐츠를 쉽게 찾고, 관련 대화에 참여할 수 있도록 함
	- sns 해시태그는 다른사람이 해시태그 검색시 내 게시물 혹은 제품이 검색되기 위해서 **최대한 비슷하게 많이 작성**하는것이 일반적
		- \#일상 https://www.instiz.net/name/39911101
		-  해시태그 상위 노출 https://in4obank.tistory.com/10
	- 또한 이벤트, 캠페인, 뉴스 등을 홍보하는 데 널리 사용
		- https://marketsplash.com/benefits-of-hashtag-campaigns/#link3
2. 해시태그 vs. 태그 vs. 카테고리
	- **해시태그(Hashtag)**: 
		- 소셜 네트워크에서 '#' 기호 뒤에 오는 단어나 문구로, 핵심 키워드나 관심 주제를 표현
		- 해시태그를 클릭하면 동일하게 태그된 다른 포스트를 검색
	- **태그(Tag)**: 
		- 웹사이트의 이미지나 텍스트와 연관된 키워드
		- 사용자가 비슷하거나 연결된 콘텐츠를 쉽게 검색
	- **카테고리(Category)**: 
		- 웹사이트 내 콘텐츠를 주제별로 구분하는 분류 시스템
		- 일반적으로 웹사이트 메뉴
	- **업무의 효율 향상을 위해 옵시디언에서 태그로 문서를 주제별로 구분**


#### 해시태그(HashTag, #) 도입 목적:

1. **효율적인 문서 검색**: 해시태그를 활용하면 필요한 문서를 신속하게 찾아볼 수 있습니다. 이는 정보 접근 시간을 대폭 줄여줍니다.
2. **문서 분류의 일관성 유지**: 공통된 해시태그를 사용함으로써 문서를 체계적으로 분류하고 일관되게 관리할 수 있습니다.
3. **팀원 간 정보 공유 촉진**: 모든 팀원이 같은 방식으로 문서에 태그를 부착함으로써 서로의 작업을 보다 깊이 이해하고 협업 효율을 높일 수 있습니다.
4. **시간별 통계 제공**: 날짜별 해시태그를 통해 주별, 월별, 연별로 각종 이슈에 대한 통계를 쉽게 확인할 수 있습니다.
5. **업체별 통계 확인**: 해시태그를 통한 업체별 검색으로, 각 업체 관련 발생한 이슈와 타임라인을 한눈에 파악할 수 있습니다.

#### (제안) 해시태그(HashTag, #) 사용 방법:

- **효율적인 문서 검색을 위해 해시태그를 카테고리와 접목**
- **HashTag 1~3 - 계층적 해시태그 사용**: 담당부서, 문서명, 날짜 등 기본정보
- **HashTag 4, 5 - 부서별 해시태그**: 각 부서별로 자주 사용하는 문서 유형(카테고리)에 대해 해시태그를 미리 정의하고 이를 반드시 지켜서 적는다.


| HashTag   | 정의                | 예시                     | 설명                     |
| --------- | ----------------- | ---------------------- | ---------------------- |
| HashTag 1 | 담당부서/문서명          | \#11/INF               | 다빛문서명칭 참고              |
| HashTag 2 | 문서명               | \#INF                  | 다빛문서명칭 참고              |
| HashTag 3 | 날짜(YYYY/MM/DD)    | \#2024/05/13           |                        |
| HashTag 4 | 핵심카테고리<br>        | \#교육<br>\#공지<br>\#제품수리 | 부서마다 정하지만, 공통내용은 서로 공유 |
| HashTag 5 | 핵심단어<br>(부서마다 상이) | \#APPLE<br>\#컨트롤러가격    | 문서의 핵심내용으로 정의          |

* **규칙**
	* 각 부서의 \*1. Project 폴더만 HashTag를 달 것
	* 영어는 모두 대문자로
	* 정해진 HashTag는 임의로 줄이거나 늘리지 않고 정해진대로 쓸 것
	* HashTag별 정의]
		* HashTag 1,2,3는 반드시 작성
		* HashTag 4는 각 부서에서 정해놓은 카테고리
		* HashTag 5는 문서의 핵심단어로 간단하게 작성

#### (추천플러그인) TagFolder
- 주요 기능:
	- **자동 분류**
		- TagFolder는 문서에 추가된 해시태그를 기반으로 자동으로 폴더 구조를 생성 
		- 해당 태그가 포함된 모든 문서를 해당 폴더로 이동
	- **동적 업데이트**
		- 문서에 새로운 태그를 추가하거나 변경할 때마다, TagFolder는 폴더 구조를 실시간으로 업데이트하여 반영
	- **검색과 정보 공유의 용이성**
		- 자동 분류는 문서 검색 시간을 대폭 감소
		- 관련 문서들이 명확하게 정리되어 있어 필요한 정보를 빠르게 검섹
		- 팀 내에서 쉽게 공유 가능
- 적용사례
	1. 부서별 검색
	2. 연도별 검색
	3. 문서별 검색
	4. 업체별 검색
	5. 제품별 검색
