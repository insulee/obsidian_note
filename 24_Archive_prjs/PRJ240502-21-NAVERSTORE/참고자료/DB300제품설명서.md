### 메모
#### 용도

   - DIBD300은 ESP32가 사용된 시리얼 to 이더넷 모듈입니다. DIBD300은 시리얼 장비가 보낸 데이터를 TCP/IP 데이터로 변환시켜 네트워크로 전송시키며, 반대로, 네트워크로부터 받은 TCP/IP 데이터를 시리얼 데이터로 변환시킨 후 시리얼 장비로 전달하는 역할을 수행합니다. 별도의 개발이 필요없이, 전원 공급 후, LAN과 Serial 케이블만 연결하면 동작하게 됩니다.

#### 규격 및 정격

![[DB300사양서_2.png]]
   - 크기 (mm) : 65(L) X 33(W) X 18(H)
   - 입력 전압 : 5V
   - 최대 소비 전력 : 200mA
   - 산업용 동작온도 : -40 ~ 85℃

#### 사용법

**여기서는 DABIT300을 아래와 같이 “랜(LAN) TO 시리얼(TTL)”로 연결/설정하는 방법을 설명한다.**
☞ [DABIT300 사용 가이드](http://www.dabitsol.com/files/DABIT300%20User%20Manual.pdf)참조

1. DABIT300의 TTL 포트와 DABIT500의 TTL 포트를 표준 연결케이블(4핀, 기본 제공)로 연결한다. 
2. DABIT300에 랜케이블을 연결하고, DABIT500에 전원을 투입한다. 
3. [DABIT Net(v9.0)](http://www.dabitsol.com/files/DabitNet_v9.0.zip) 을 다운로드하여, 사용자 네트워크 환경에 맞게 통신 정보를 설정/변경한다. 
		☞  [DABIT Net 사용  가이드](http://www.dabitsol.com/files/DABITNet%20Guide.pdf)참조
4. 시리얼 통신 포트와 통신 속도를 설정한다. 
		☞ 기본값 : TTL 포트, 115200bps
5. 사용자의 네트워크 환경에 맞게 “통신 모드, IP 정보” 등을 설정한다.

#### 기타 주요 기능

**1. 하드웨어 초기화 기능**
   - 전원이 투입된 상태에서 DABIT300 뒷면의 리셋 단자를 일자 드라이버로 쇼트 시키면, 설정 정보들이 다음과 같이 초기화된다.
	   ☞ IP주소 : 192.168.0.202, 포트: 5000번, TCP서버모드
	   ☞ 시리얼통신 : TTL(J3), 115200bps
![[DB300사양서_4.png|500]]

**2. 원격 리셋 기능(UDP 통신)**
   - TCP 접속 중에 비정상적으로 연결이 끊어지거나, 기타 사유로 CPU 리셋이 필요한 경우, UDP통신으로 “RESET  DIBD-ALL” 패킷을 전송하면, DABIT300이 리셋된다.  UDP 통신의 Port는 “5108”로 고정이다. 
	   ☞ [소켓 통신 가이드](http://www.dabitsol.com/files/%EB%8B%A4%EB%B9%9B%20%EC%A0%84%EA%B4%91%ED%8C%90%20%EC%BB%A8%ED%8A%B8%EB%A1%A4%EB%9F%AC%EC%9D%98%20%EC%86%8C%EC%BC%93%20%ED%86%B5%EC%8B%A0%20%EA%B0%80%EC%9D%B4%EB%93%9C.pdf) 
   - 리셋 패킷을 UDP Broadcast 로 전송하면, 네트워크망에 존재하는 모든 DABIT300 컨트롤러들의 CPU 가 리셋된다.

**3. 자동 리셋 기능**
   - TCP 접속 중에 비정상적인으로 연결이 끊어질 때(클라이언트 전원차단 등 사유) 서버측(DABIT300)은 연결이 끊어진 상태를 인지하지 못함. DABIT300은 이런 상황에서 클라이언트가 다시 연결을 시도할 때, 자동 리셋되고, 다시 한번 더 연결을 시도하면, TCP 통신이 정상으로 재개된다.

**4. 전원 분리 투입 방법**
   - 시리얼 통신 전압 레벨이 5V를 아래와 같이 전원 케이블을 분리하여 연결한다. 
   - 시리얼통신포트(예, TTL)에는 “TX, RX, Ground”만 연결한다. (기본 케이블 4핀 중에 “5V” 케이블만 제거함) 
   - 사용하지 않는 포트에 “5V, Ground” 를 연결한다. 연결선은 “USB 커넥터, 몰렉스(2핀) 커넥터” 형태로 요청시 별도 제공가능.
