## 1. 고객지원, 기술지원 관리 방식의 변화

**배경**

**핵심 문제**
- 업무는 업체별로 발생하지만, 문서는 성격별로 분산 저장되어 업체 관리의 비효율 발생
- 업체 문의 대응 시 AS, 기술지원, 회의록, 제안서 등을 각 폴더에서 개별 검색
- 업체별 이력을 시간순으로 파악하기 어려움

**분류 구조의 변화**
- 기존: `문서성격 > 연월 > 업체` 순 분류 → 업체 히스토리 파편화
- 개선: `업체 > 연월 > 문서성격` 순 분류 → 업체별 타임라인 확보

---

**기존 방식 (문서 성격별 분류)**
```
23_기술지원/
  ├── 2월/
  │   └── TCS250205-업체명.md
  └── 9월/
      └── TCS250903-업체명.md

24_AS업무/
  ├── 2월/
  │   └── A_S250205-23-업체명.md
  └── 11월/
      └── A_S251103-업체명.md
```
- 문서 성격별로 폴더를 만들고 연월 기준으로 분류
- 간단한 고객지원 사항은 파편적으로 기록


**새로운 방식 (업체별 통합 관리)**
```
23_업체관리/
  └── 업체명/
      ├── !업체명.md (대시보드)
      ├── 251103-AS-업체명.md
      ├── 250827-TS-업체명.md
      ├── 251014-DOC-업체명.md
      └── 251114-MOM-업체명.md
```
- 업체별로 폴더를 만들고 관련 문서를 모두 한 곳에 모음
	- AS: 제품 AS 처리 건
	- TS: 기술지원 문서
	- DOC: 공식 문서, 내부 문서 (업체 전달용 가이드, 제안서 등)
	- MOM: 업체 회의록 
- 일일노트의 태그로 자동 집계되어 시간순 히스토리 확인 가능

**차이점**
- **문서 분류 기준(성격별) ≠ 업무 발생 단위(업체별)**
- **기존 방식**: 업체 관련 문서를 찾으려면 기술지원, AS업무, 회의록 등 여러 폴더를 검색
- **새로운 방식**: `23_업체관리/업체명/` 폴더 하나에서 모든 이력 확인

- AS 파일: `YYMMDD-AS-업체명.md`
- TS 파일: `YYMMDD-TS-업체명.md` 또는 `YYMMDD-TS-업체명-내용.md`
- DOC 파일: `YYMMDD-DOC-업체명.md` 또는 `YYMMDD-DOC-업체명-내용.md`

## 3. 업무 워크플로우

- **간단한 CS 문의**: `#cs` 태그로 일일노트에만 기록
- **상세 기술문서**: `#ts` 또는 `#doc` 태그로 업체 폴더에 문서 저장
- **AS 처리**: `#as` 태그로 업체 폴더에 문서 저장
- **업체회의록**: `#mom` 태그로 업체 폴더에 문서 저장

### Case 1: 간단한 CS 문의/대응
```
일일노트에 기록만
↓
- [ ] #cs #제이엘유 402 펌웨어 테스트
↓
23_업체관리/제이엘유/!제이엘유.md 에 진행중인 업무에 집계됨
↓
완료 체크
↓
완료된 업무에 집계됨
```

### Case 2: 기술지원 문서(TS) 또는 공식 문서(DOC) 작성 
```
1. 일일노트에 기록
   - [ ]  #doc #제이엘유 [[251014-DOC-제이엘유-TTL연결시스템구성제안]]

2. TS 문서 작성
   → 23_업체관리/제이엘유/ 폴더에 저장
   → 파일명: 251014-DOC-제이엘유-TTL연결시스템구성제안.md
   
3. 일일노트에 완료 표시
   - [x] #doc #제이엘유 [[251014-DOC-제이엘유-TTL연결시스템구성제안]]

4. 23_업체관리/제이엘유/!제이엘유.md에 완료된 업무에 집계
```

### Case 3: AS 처리
```
1. AS 문서 작성
   → 23_업체관리/제이엘유/ 폴더에 저장
   → 파일명: 251103-AS-제이엘유.md
   
2. 일일노트에 기록 후 체크
   - [x] #as #제이엘유 [[251103-AS-제이엘유]]

3. 23_업체관리/제이엘유/!제이엘유.md에 완료된 업무에 집계
```

## 4. 업체별 관리 대시보드(`!업체명.md`) 활용

#### 자동 표시 내용
1. **진행중인 업무** - 업무노트에서 미완료 태그 자동 수집 (dataview)
2. **완료된 업무** - 최근 완료 이력 자동 수집 (dataview)

**활용 방법**: 
- 업체 연락이 오면 `!업체명.md`를 열어 진행중/완료 업무 즉시 확인
- CS, TS, DOC, AS, MOM 태그가 모두 시간순으로 표시되어 히스토리 추적 가능
- 필요시 dataview 아래에 중요 이력을 직접 작성 가능
- 같은 폴더 내 TS, DOC, AS, MOM 문서 파일도 함께 확인

#### 예시: `!제이엘유.md`

```markdown
> [!Abstract] 업체 담당자 및 업체 정보
> 담당자: 김부영 대표님
> 연락처: 010-6404-1192

## 진행중인 업무
(dataview 쿼리)

## 완료된 업무
(dataview 쿼리)
```

**완료된 업무에 표시되는 실제 예시**:
- 2025-11-04(화): `#as #제이엘유` [[251103-AS-제이엘유]]
- 2025-10-21(화): `#cs #제이엘유` 402 펌웨어 테스트
- 2025-10-14(화): `#doc #제이엘유` [[251014-DOC-제이엘유-TTL연결시스템구성제안]]

## (참고) 

### 핵심 태그 4가지

| 태그     | 용도                 | 파일명 형식            |
| ------ | ------------------ | ----------------- |
| `#cs`  | 고객 대응 (간단한 기술문의)   | -                 |
| `#ts`  | 기술지원 (상세 기술문서)     | 연연월월일일-TS-업체명.md  |
| `#doc` | 문서 작성 (내부/업체 전달문서) | 연연월월일일-DOC-업체명.md |
| `#as`  | AS 처리 건            | 연연월월일일-AS-업체명.md  |
| `#MOM` | 업체 회의록             | 연연월월일일-MOM-업체명.md |

**중요 변경사항**:
- `23_기술지원/` 폴더에 저장하던 기술문서, `24_AS업무/` 폴더에 저장하던 AS문서도 업체 폴더에 저장
- `#cs`는 간단한 문의만 일일노트에 기록
- `#ts`는 기술지원 상세 문서 작성시 사용
- `#doc`는 공식 문서 또는 내부 문서 작성시 사용
- 파일명 규칙 변경: 
	- `A_S251103-업체명.md` → `251103-AS-업체명.md`
	- `TCS251103-업체명.md` → `251103-TS-업체명.md`

### 업체명 태그
모든 업무에 **업체명 태그**를 함께 사용합니다.
```markdown
- [ ] #cs #제이엘유 402 펌웨어 테스트 (일일노트에만)
- [x] #ts #우시옵토 모듈 노이즈 분석 [[250410-TS-우시옵토-모듈노이즈발생]]
- [x] #doc #제이엘유 TTL 연결 가이드 [[251014-DOC-제이엘유-TTL연결시스템구성제안]]
- [x] #as #제이엘유 컨트롤러 점검 [[251103-AS-제이엘유]]
```
---
### 파일 태그 추가 규칙

모든 TS, DOC, AS 파일에는 반드시 상단에 태그를 추가해야 합니다.

### 기본 형식
```yaml
---
tags:
  - "#태그종류"  (ts, doc, as 중 하나)
  - "#업체명"
---
```

### share_link가 있는 경우
```yaml
---
share_link: https://share.note.sx/...
share_updated: 2025-11-03T10:33:44+09:00
tags:
  - "#doc"
  - "#우시옵토"
---
```

### 태그 추가 예시
- TS 파일: `250410-TS-우시옵토-모듈노이즈발생.md` → `#ts` + `#우시옵토`
- DOC 파일: `251014-DOC-제이엘유-TTL연결시스템구성제안.md` → `#doc` + `#제이엘유`
- AS 파일: `251103-AS-제이엘유.md` → `#as` + `#제이엘유`

### 폴더 정렬 설정

### Custom File Explorer Sorting 플러그인
23_업체관리 폴더에서 **폴더와 파일을 섞어서 정렬**하려면:

1. 플러그인 설치: `Custom File Explorer Sorting`
2. `22_Area/23_업체관리/` 폴더에 `sortspec.md` 파일 생성
3. 정렬 규칙 설정:

```yaml
---
sorting-spec: |
  order-asc: a-z
---
```

자세한 내용: [[SRA251104-Custom File Explorer Sorting]]
