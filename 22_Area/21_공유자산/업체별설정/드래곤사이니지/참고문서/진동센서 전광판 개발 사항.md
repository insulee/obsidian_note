### 1. 기본 사양 및 운영 정책

1. **통신 인터페이스**  
   - RS-232, 9600 bps 
   - 데이터 프레임: 8비트 데이터, 패리티 없음(N), 1비트 스톱(8-N-1)

1. **데이터 표출**  
   - Z축 값만 표시
   - 전광판 구성: 1단 4열 레이아웃, 단위(dBV)  
   - 예시 이미지
      - ![[드래곤_진동전광판.png|300]]

1. **데이터 저장** 
   - 기록 주기: 5분 간격 데이터 저장  
   - 파일 분할: 월별 단위 TXT 파일 생성 
   - 파일명 규칙
     - 진동값: `YYYY-MM-V.txt`  
     - 소음값: `YYYY-MM-A.txt`  
     - 미세먼지값: `YYYY-MM-M.txt`

### 2. 명령어 형식


- HEX
``` 
50 33 0D 0A
```
- ASCII
``` 
P3<CR><LF>
또는
P3\r\n
```

- **P3**: Z축 데이터 요청 명령 (P: 0x50, 3: 0x33)
- **CR**: Carriage Return (0x0D)
- **LF**: Line Feed (0x0A)
- 참고
	- P0: X, Y, Z축 전체 데이터 요청
	- P1: X축 데이터 요청
	- P2: Y축 데이터 요청
	- P3: Z축 데이터 요청


### 3. 응답 형식

- `<Axis><value><CR>` 구조
- **Axis**:
    - `Z` (0x5A)
- **value**
	- 소수점(`2E`)을 포함한 5바이트
	- 예: `113.5` -> `31 31 33 2E 35`
- **CR(0x0D)**: 줄 구분 제어문자 
- **예시 Z113.5 (Hex)**:
    ```
    5A 31 31 33 2E 35 0D
    ```
