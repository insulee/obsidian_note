## 1. 프로토콜 기본 구조

### 1.1. 프로토콜 명령 패킷 구성
#### 명령 패킷: DLE + STX + DST + LEN + CMD + DATA + DLE + ETX

| 구분       | 크기                  | 설명                                        |
| -------- | ------------------- | ----------------------------------------- |
| **DLE**  | 1 byte              | **HEX 프로토콜의 시작 코드: `0x10`**               |
| **STX**  | 1 byte              | **HEX 프로토콜의 시작 코드: `0x02`**               |
| **DST**  | 1 byte              | **전광판 주소**                                |
|          |                     | - 싱글모드(1:1 통신) : `0x00`                   |
|          |                     | - 멀티모드(1:n 통신) : `0x01` ~ `0x0F` (RS-485) |
|          |                     | - 마스터 주소 : `0x00`                         |
| **LEN**  | 2 byte              | **CMD + DATA 크기**                         |
|          |                     | - 하위 12bit : 데이터 길이 (최대 4096개)            |
|          |                     | - 상위 3bit : 특수목적 패킷 용도                    |
| **CMD**  | 1 byte              | **명령 코드**                                 |
|          |                     | - 메시지 표시 명령 (`0x94`)                      |
|          |                     | - 특수 기능 명령                                |
| **DATA** | {16 + (2 × n)} byte | **메시지 데이터**                               |
|          |                     | - 표시 속성 (16 byte)                         |
|          |                     | - 문자 색상 (n byte)                          |
|          |                     | - 문자 데이터 (n byte)                         |
|          |                     | - ASCII (영문/숫자) : 1byte, 한글/유니코드 : 2byte  |
| **DLE**  | 1 byte              | **HEX 프로토콜의 종료 코드: `0x10`**               |
| **ETX**  | 1 byte              | **HEX 프로토콜의 종료 코드: `0x03`**               |

### 1.2. 프로토콜 응답 패킷 구조

#### 응답 패킷: DLE + STX + DST + LEN + CMD + DATA + DLE + ETX

| 구분       | 크기     | 설명                                  |
| -------- | ------ | ----------------------------------- |
| **DLE**  | 1 byte | **HEX 프로토콜의 시작 코드: `0x10`**         |
| **STX**  | 1 byte | **HEX 프로토콜의 시작 코드: `0x02`**         |
| **DST**  | 1 byte | **전광판 주소: `0x00`**                  |
| **LEN**  | 2 byte | **CMD + DATA 크기**                   |
| **CMD**  | 1 byte | **명령 코드**                           |
| **DATA** | 1 byte | **수신 받은 패킷에 대한 응답**                 |
|          |        | - `0x00`: 정상 처리                     |
|          |        | - `0x10`: No Command (없는 명령어)       |
|          |        | - `0x20`: No Function (기능 미지원)      |
|          |        | - `0x40`: Error Rx Data (데이터 범위 초과) |
|          |        | - `0x80`: 기타 오류                     |
| **DLE**  | 1 byte | **HEX 프로토콜의 종료 코드: `0x10`**         |
| **ETX**  | 1 byte | **HEX 프로토콜의 종료 코드: `0x03`**         |

## 2. 메시지 패킷 구조
### 2.1. 메시지 패킷 구조

- 표출 예시
	- ![[30_GA_Team/20_Marketing_Team/23_Resource/Attachment/HEX1.png|500]]
	- 2단 6열(96x32픽셀)
	- 문구: `rgybmcw AaBb가나다각난닫`
	- 문구별 색상
		- `r`: 빨간색(Red)
		- `g`: 초록색(Green)
		- `y`: 노란색(Yellow)
		- `b`: 파란색(Blue)
		- `m`: 자주색(Magenta)
		- `c`: 청록색(Cyan)
		- `w`: 하얀색(White)
		- `AaBb가나다각
		- `난`: 파란색(Blue)
		- `♥`: 하얀색(White)
	- 통신연결: Serial 통신
	- 전송 패킷
		- DLE~CMD: `10 02 00 00 41 94` 
		- DATA(표시속성, 16byte): `00 00 63 01 00 03 01 01 00 0A 62 00 00 00 00 00` 
		- DATA(문자색상, 12byte): `01 02 03 04 05 06 07 07 07 07 07 07` (rg~Aa)
		- DATA(문자색상, 12byte): `07 00 07 00 07 00 17 00 34 00 07 00` (가~♥)
		- DATA(문자코드, 12byte): `72 67 79 62 6D 63 77 20 31 32 41 61` (rg~Aa)
		- DATA(문자코드, 12byte): `B0 A1 B3 AA B4 D9 B0 A2 B3 AD E0 02` (가~♥)
		- DLE, ETX: `10 03`
	- 응답 패킷
		- DLE~CMD: `10 02 00 00 02 94` 
		- DATA(응답): `00` 
		- DLE, ETX: `10 03`

#### (참고) 화면 채우기 색상 코드

- **R, G, B 값은 각각 비트**로 표현
    - R: 하위 3비트 (`00000111`)
    - G: 중간 3비트 (`00111000`)
    - B: 상위 2비트 (`11000000`)
- 혼합된 색상은 각 비트가 OR 연산으로 결합됩니다.
    - 예: 노란색 (Y) = 빨간색 (R) + 초록색 (G)
        - `00000111` (R) | `00111000` (G) = `00111111` (Y)
    - 예: 흰색 (W) = 빨간색 (R) + 초록색 (G) + 파란색 (B)
        - `00000111` (R) | `00111000` (G) | `11000000`(B)= `11111111` (W)

| **색상**  | **구성 (R, G, B)** | **Binary** | **HEX** |
| ------- | ---------------- | ---------- | ------- |
| 검정색 (B) | 없음               | 0000 0000 | 0x00    |
| 빨간색 (R) | R만 활성화           | 0000 0111 | 0x07    |
| 초록색 (G) | G만 활성화           | 0011 1000 | 0x38    |
| 노란색 (Y) | R + G 혼합         | 0011 1111 | 0x3F    |
| 파란색 (B) | B만 활성화           | 1100 0000 | 0xC0    |
| 자주색 (M) | R + B 혼합         | 1100 0111 | 0xC7    |
| 청록색 (C) | G + B 혼합         | 1111 1000 | 0xF8    |
| 흰색 (W)  | R + G + B 혼합     | 1111 1111 | 0xFF    |
