### 1. 문의 개요
- **현상**: 아이파킹 → 전광판 구간에서 `[0/1]` 패턴의 패킷이 반복 수신되면서 정상 제어 명령이 지연/차단됨
- **관련 로그**: `22_Area/23_업체관리/우시옵토/attachment/전광판이상패킷.txt` 
- 수신 바이트는 ASCII `/`, `0`, `]`, 등으로 구성되어 `[0/1]` 문자열이 무한 반복되는 형태. 
```startLine:36:endLine:47:22_Area/23_업체관리/우시옵토/attachment/전광판이상패킷.txt
"2025-11-04 14:22:24.987 ... : 2f"
"2025-11-04 14:22:24.992 ... : 0a"
"2025-11-04 14:22:25.015 ... : 30"
"2025-11-04 14:22:25.014 ... : 5b"
"2025-11-04 14:22:25.017 ... : 31"
"2025-11-04 14:22:25.044 ... : 5b"
"2025-11-04 14:22:25.044 ... : 30"
"2025-11-04 14:22:25.045 ... : 2f"
... (중략)
```
```34957:34965:22_Area/23_업체관리/우시옵토/attachment/전광판이상패킷.txt
"2025-11-04 14:27:24.038 ... : 2f"
"2025-11-04 14:27:24.040 ... : 30"
"2025-11-04 14:27:24.041 ... : 5d"
"2025-11-04 14:27:24.042 ... : 0a"
// 이후 "[0/1]\n" 패턴 반복
```

| Hex 값 | ASCII | 설명 |
| --- | --- | --- |
| `0x5b` | `[` | 페이지 메시지 시작 |
| `0x30` | `0` | 현재 페이지 index (0) |
| `0x2f` | `/` | 구분자 |
| `0x31` | `1` | 총 페이지 수 (1) |
| `0x5d` | `]` | 페이지 메시지 종료 |
| `0x20` | 공백 | 일부 구간에서 삽입 |
| `0x0a` | 개행 | 디버그 줄바꿈 |

### 2. 문의 요약
| 구분  | 내용                                                                                                 |
| --- | -------------------------------------------------------------------------------------------------- |
| 1   | 위즈넷 LAN-to-Serial 컨버터 설정 화면에서 `Enable Serial Debug Mode`가 체크된 이유와 동작 방식 확인 요청                      |
| 2   | 수거한 전광판 컨트롤러는 디버그 모드 활성화 되어 있었음. 수거 후 테스트할 때 현장과 달리 `[0/1]` 패킷이 보이지 않는 이유와, 디버그 모드에서 반복 송신되는 조건 문의 |

### 3. 회신 내용
- **위즈넷 LAN-to-Serial 컨버터**
	- 이더넷–RS232 변환기로 위즈넷 제품
	- 위즈넷 프로그램에서 `Enable Serial Debug Mode` 체크 시, 변환기를 통과하는 모든 시리얼 데이터를 PC 화면에 그대로 표시
	- 체크/해제 후 `Setting` 버튼을 눌러야 장치에 적용됨
	- 체크 상태로 조회되면 컨버터 쪽 디버그가 활성화된 상태
	- **다빛솔루션 컨트롤러 내부 설정에는 영향을 주지 않음**

- **수거한 컨트롤러의 디버그 활성화**
	- 디버그 모드가 Enable 시 페이지 상태 패킷을 지속 출력
	- `[0/1]`, `[1/3]` 등 포맷은 `현재 페이지/설정된 총 페이지 수`를 의미
	- 현재 디버그 모드가 `enable3`으로 되어 있어 `[0/1]` 패킷이 출력 됨
	- 디버그 모드가 비활성화(Disable)되면 해당 패킷은 출력되지 않음
	- 디버그 모드를 활성화하는 패킷이 전송되지 않는 한, 디버그 모드가 활성화 되지는 않음
	- 위즈넷의 프로그램에서 Enable Serial Debug Mode를 체크하면 해당 컨버터의 디버그 모드가 활성화 되는 것으로, 다빛솔루션 컨트롤러의 디버그 모드가 disable에서 Enable3으로 변경되지는 않음
	- 개발팀 또는 현장에서 테스트 중에 디버그 모드를 활성화 했다가 disable로 바꾸지 않고 조립된 경우가 종종 있음


- **시뮬레이터 재현 절차 및 패킷 정보**
	1. **재현 절차 (다빛 프로토콜 시뮬레이터)**
		1. `설정 및 제어 → 페이지 메시지 설정` 이동
		2. `메시지 초기화: 전체 → 전송`
		3. `총개수 등록: 1개 → 전송`
		4. `[0/1]` 패킷이 대량 송신되는 것을 확인 가능
		5. 페이지 메시지 1번을 전송하면 설정된 유지 시간에 따라 `[0/1]` 패킷이 계속 송신되는지 확인 가능
	2. **패킷 상세 및 현상**
		- `![006199!]` → `![006001!]` 순서만 전송해도 `[0/1]` 패킷이 지속 출력됨
		- 페이지 메시지 1번 데이터를 전송하면, 설정된 유지 시간에 따라 `[0/1]` 패킷 송신 주기 결정

| 단계                | ASCII                | HEX                             |
| ----------------- | -------------------- | ------------------------------- |
| 페이지 메시지 초기화       | `![006199!]`         | `10 02 00 00 02 4B 80 10 03`    |
| 페이지 메시지 개수 등록(1개) | `![006001!]`         | `10 02 00 00 02 4C 01 10 03`    |
| 페이지 메시지 1번 전송     | `![001/P0000/....!]` | 레거시: `54 00` / 최신 프로토콜: `94 01` |
