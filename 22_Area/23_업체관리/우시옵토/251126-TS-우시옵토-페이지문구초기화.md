# 고정문구 초기화 현상 - 문의 및 해결 방법

## 📋 고객 문의 내용

### 업체 정보
- 문의 업체: 우시옵토
- 납품처: 아이파킹
- 생산/조립 시기: 2024년 6월
- 펌웨어: 다빛 펌웨어 (버전 확인 안됨)

### 접수 증상

고정문구가 N시간 이후 초기화 되는 현상

- 정확한 시간은 알 수 없으나 고정문구 상태가 퇴근 후 다음날 초기화된 상태로 발견
- 다른 문구로 변경하지 않는 한 무한 노출(고정문구)되는 것인가요?

### 참고사항

고정문구 = 우시옵토에서 출고 시 고정된 페이지 문구로 납품함 (통신, 전광판 모듈 테스트 인증용)

## 💡 답변 내용

문의하신 고정문구 초기화 현상에 대해 답변드립니다.

### ○ 원인 파악을 위한 확인 사항

먼저 아래 내용을 확인해주시기 바랍니다.

**1. 초기화된 문구 확인**
   - 우시옵토에서 납품 시 설정한 페이지 문구가 그대로 유지되다가 초기화된 것인지?
   - 아니면 아이파킹에서 문구를 임의로 변경한 후 초기화된 것인지?

**2. 문구 변경 시 전송 방식 확인 (아이파킹에서 문구를 변경한 경우)**
   - 페이지 메시지(![001/!])로 전송했는지?
   - 실시간 메시지(![000/!])로 전송했는지?

### ○ 가능한 원인 및 해결 방법

**[경우 1] 우시옵토 납품한 페이지 문구가 초기화된 경우**

→ 디버깅 모드가 Enable 상태일 가능성이 있습니다.

확인 및 해결 방법:
1. 다빛솔루션 프로그램 실행
2. 보드기능설정 → "2. Sub command" 선택
3. "읽기" → "전송" 클릭
4. "3. Debugging Method" 확인
   - Enable인 경우 → Disable로 변경 후 전송

Disable이라면 고정문구가 초기화 됐을때 컨트롤러의 송수신 데이터를 확인해야 함

**[경우 2] 아이파킹에서 페이지 메시지가 아닌 실시간 메시지로 전송한 경우**

→ 전원이 껐다 켜지면 실시간 메시지는 RAM에 저장되어 있어 초기화됩니다.

해결 방법:
- 문구를 페이지 메시지(![001/!])로 재전송해주세요.
- 페이지 메시지는 ROM에 저장되어 전원이 껐다 켜져도 유지됩니다.
