### 표출 화면 스케줄 확인 패킷 활성화
#### **목표**
- 전광판에서 스케줄에 등록된 여러 파일을 순차적으로 표출.
- 각 파일의 **표출 효과 종료** 시, **시리얼 통신**을 통해 **확인 패킷**을 전송받음.
#### **설정**
- 펌웨어: `DIBD600T_08C_04R016C_V3.0.5-UM.bin`
- 보드기능설정 > 3. Debugging Method > `Disable`
---
### **Protocol 상세**
#### 1. **전송 패킷 (Transmit Packet)**
- 형식: `![0057 a b cde!]`
    - **`a` 값**: 스케줄 확인 패킷 활성/비활성 제어
        - `0`: 비활성화
        - `1`: 활성화
	- **`b` 값**: 통신 포트 선택
		- `0`: RS232 통신
		- `1`: TTL 통신
	- `cde`값: 스케쥴 확인 패킷 전송 후 지연 시간(0~255ms)
	
---
#### 2. **응답 패킷 (Response Packet)**
- **응답 패킷 1 (상태 확인)**:
    - 성공: `![00570!]`
    - 실패: `![0057F!]`
- **응답 패킷 2 (스케줄 진행 정보)**:
    - 형식: `![0057 abc def ghi!]`
        - **`abc`**: 스케줄에 등록된 **전체 파일 수** (예: `003` → 총 3개 파일)
        - **`def`**: 현재 표출 효과가 종료된 **파일 번호** (예: `002` → 2번 파일)
        - `ghi`: 스케줄 확인 패킷 전송 후 지연 시간 (0~255ms)
            - 스케줄 확인 패킷은 현재 화면의 스케줄 종료 후 전송됩니다. 단말기는 이를 수신한 뒤 **다음 명령 패킷**을 송신하지만, 대기 시간이 없으면 전광판 컨트롤러가 해당 패킷을 수신하기 전에 다음 화면이 먼저 표출될 수 있습니다. 이를 방지하기 위해 최대 **255ms의 대기 시간(cde 값)** 설정이 필요합니다.

---
### **주요 특징**
1. **활성화/비활성화 상태**
    - 기본적으로 스케줄 확인 패킷은 **비활성화 상태**.
    - 송신 패킷 `![00571!]`을 통해 **활성화** 가능.
2. **공장 초기화 시 비활성화**
    - 공장 초기화(factory reset)가 실행되면 스케줄 확인 패킷은 비활성화 상태로 복귀.
    - 초기화 이후 활성화 상태를 설정하려면 반드시 송신 패킷을 전송해야 함.

---
### **통신 흐름 예시**
1. **스케줄 활성화 요청**
    - 송신: `![0057 1 0 100!]`
        - `1`: 스케쥴 확인 패킷 활성화
        - `0`: RS232 통신
    - 수신(응답):
        - 성공 시: `![00570!]`
        - 실패 시: `![0057F!]`
2. **스케줄 진행 확인**
    - 전광판 스케쥴 전송 예시(경찰청 접이식)
        - `![0020002 014 023!]`
        - 28개의 배경이미지 중 2번째 14번째 23번째 표출
    - 전광판에서 각 파일의 표출 효과 종료 후 자동 전송:
        - 예: `![0057 028 001 100!]`
            - 등록된 총 파일 수: `028`
            - 현재 종료된 파일 번호: `001` -> 2번째 이미지
            - 지연 시간: `100`ms
        - 이후 순차적으로 전송
            - `![0057 028 013 100!]` -> 14번째 이미지
            - `![0057 028 022 100!]` -> 23번째 이미지
