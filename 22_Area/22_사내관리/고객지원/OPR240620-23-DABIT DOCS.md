
#### DABIT DOCS 제작 목적

- **고객**:  고객이 자주 겪는 문제들을 체계적으로 정리한 문서를 제공함으로써, 고객들이 더 효과적으로 제품 활용 가능
- **다빛솔루션**: 고객서비스와 기술 지원에 소요되는 시간과 에너지를 절약하여 반복적인 문제 해결 대신 더 복잡한 문제에 집중

많은 기업들이 DOCS 페이지를 만들어서 고객들에게 정보를 제공

#### 홈페이지에서 DABIT DOCS로

1. **전환 과정**
    - 홈페이지의 모든 콘텐츠를 DABIT DOCS로 이전할 계획
    - DABIT DOCS는 옵시디언 기반으로, Publish 기능을 이용해 수정과 배포가 용이
2. **DABIT DOCS의 역할**
	- DABIT DOCS는 매뉴얼과 전문 자료를 한 곳에 모아두어 고객 응대 시 필요한 정보를 손쉽게 검색
	- 직원들은 DABIT DOCS의 새로운 업데이트 내용을 확인하고, 이를 통해 고객 응대 시 정확한 정보를 제공

#### NOTE와 DABIT DOCS 문서 재배치 및 관리

DABIT DOCS와 NOTE를 명확하게 구분하여, 효율적인 정보 관리와 접근성을 높일 예정

1. **DABIT DOCS**
    - 현재 홈페이지에 있는 내용을 DABIT DOCS로 이전하면서 최신 정보로 업데이트 하여, 고객들에게 정확한 정보를 제공
	- 고객이 자주 질문하는 내용 중 공개 가능한 정보는 DABIT DOCS로 이동한 뒤 NOTE에서 삭제하여 DABIT DOCS 내에서만 검색 가능
        - 예: 제품별 최대 표출 가능 모듈 수, 제품별 가격, 각종 설정 방법들 (표출신호설정, 전광판크기설정, 하드웨어 리셋방법 등)
2. **Obsidian NOTE**
    - 고객에게 공개할 필요없는 회사 내부의 기술 자료, 사내 교육 자료, 특정 업체 관련 사항들, 재고 수량, 주간/일간 노트 등을 관리
    - 브랜드마케팅팀을 제외한 나머지 팀은 기존의 작성 방식을 유지하되, DABIT DOCS에 적합한 내용은 브랜드마케터가 선정 후 해당 팀과 조율을 통해 DABIT DOCS로 이동
3. **DABIT DOCS 내용 변경 및 관리**
    - DABIT DOCS에 있는 내용 변경이 필요할 경우, 브랜드마케터에게 요청하여 수정
    - 변경된 내용은 NOTE에 기록하여, 주간회의 때 직원들에게 변경된 내용을 공유
4. **AS/기술문의**
    - 브랜드마케터와 소프트웨어엔지니어는 AS/기술문의 대응시 NOTE의 A_S에 각자 문서화
    - 브랜드마케터는 A_S의 문서화된 자료 중 민감정보를 삭제한 문제/해결 위주로 DABIT DOCS에 업로드하여 기술지원에 활용

#### DABIT DOCS 제작 플랜
- 소프트웨어엔지니어 입사전까지 전까지 DABIT DOCS의 기본 내용 제작
- 신입 교육 중 필요한 내용 및 신규 내용 추가
- 홈페이지 내용을 최신의 내용으로 알기 쉽게 정리
- 최종으로 DABIT DOCS가 다빛솔루션 홈페이지를 대체
