> **반복 업무 자동화, 복잡한 문제 필터링, 신속한 사람 연결**
---
### 1. 챗봇의 기본 목적
- **반복된 질문 자동화**
    - 자주 반복되는 단순 질문(제품 문의, 사용 방법, 다운로드 등)을 챗봇이 대신 처리
    - 업무 부하 감소 → 직원들이 보다 생산적인 업무에 집중 가능
- **24시간 즉시 응답 제공**
    - 고객이 언제 질문해도 빠르게 답변 가능
    - 기술지원 접근성 향상

---

### 2. LLM(대규모 언어모델) 보강

- 기존 시나리오 챗봇은 복잡한 예외 상황에는 한계
- **GPT/Gemini 등 LLM 활용**
    - 설명이 모호하거나 예외사항이 많은 경우에도 **맥락 파악 및 답변** 가능
    - 사람이 직접 개입하지 않아도 해결 가능한 범위를 넓힘
- LLM이 이해하지 못하는 경우, **상담원 또는 직원 연결**로 전환

---
### 3. 카카오톡 챗봇과 LLM 활용

**(1) 카카오톡 챗봇의 기본 운영 구조 - 버튼 기반 응답**

- 카카오톡 채널은 대부분 버튼형 챗봇을 운영 
- 긴 자연어 답변보다는 **빠른 선택과 명확한 프로세스 안내**가 고객에게 편리
- 또한, 긴 문장으로 설명하는 대화형 응답은 카카오톡 UI 특성상 가독성이 떨어지고, 고객이 원하는 정보를 찾는 속도도 늦어지므로 잘 활용되지 않음


**(2) LLM(GPT/Gemini) 활용 포인트**
- **자연어 이해(의도 파악) + 발화 연결**
    - 고객이 원하는 버튼이 없어서 `펌웨어 업데이트 하고싶어`처럼 자유롭게 말하면
    - LLM이 이를 해석 → 내부적으로 `펌웨어 업데이트` 시나리오 실행 → 버튼 안내
- **대화형으로 길게 설명하기보다는 맥락 정리 및 시나리오 매핑에 초점**
    - GPT처럼 장황한 답변은 부정확한 정보와 가독성 문제로 혼란 야기
    - 질문 → 의도 인식 → 알맞은 메뉴 or 블록 연결 방식
- **예외 또는 모호한 상황 대응**
    - 고객 질문이 불완전하거나 애매할 경우에도 LLM이 **핵심 키워드 추출**
    - 챗봇이 적합한 흐름으로 자연스럽게 연결
    - 예: 
        - LED 컨트롤러가 네트워크 연결이 안되는데? 
        - `네트워크 문제 해결` 시나리오로 연결

---
### 4. 기업들이 쓰는 운영 전략

- **챗봇은 100% 사람을 대체하지 않음**
    - 챗봇은 최종 해결자가 아니라, **1차 필터 + 보조 도구**로 활용
    - 고객 질문을 **분류/정리/기본 대응** 후 필요 시 사람에게 넘기는 전달
- **버튼/시나리오 중심 운영**
    - 실제 운영 기업들은 고객이 빠르게 원하는 정보를 찾도록 **버튼형 UI**를 채택
    - 긴 대화형 설명을 제공하기보다는, **짧고 명확한 버튼 선택지**가 고객 경험 만족도가 더 높음
    - 잘못된 긴 답변이 주는 리스크(정보 오류, 불필요한 해석)를 피하기 위해 LLM은 백엔드에서 **질문 → 의도 인식 → 버튼 매핑**에만 활용

- **LLM의 보조 역할**
    - 고객이 자유롭게 입력한 문장을 **의도(Intent)**로 해석
    - 내부적으로 해당되는 버튼/시나리오 블록 실행 → **자연어 질문도 버튼형 UX로 연결**
    - 예:
        - 고객: “어제 저녁에 계산했는데 왜 돈이 빠져나간 게 안 보여요?”
        - LLM: `intent=transaction_history` 인식
        - 챗봇: <거래 내역 확인하기> 버튼 제시
    - 예:
        - 고객: “편의점에서 긁었는데 자꾸 결제가 안 된다네요”
        - LLM: `intent=card_payment_issue` 인식
        - 챗봇: <결제 오류 확인하기> 버튼 제시
- **답변 실패 대비**
    - 복잡하거나 불명확한 질문은 무리해서 답변하지 않고 상담사 연결
    - 카카오뱅크: 상담사 연결 버튼 제공
    - 현대백화점: 고객센터 연결 경로 전환

> [!abstract] **간단한 문제 = 즉시 해결 / 복잡한 문제 = 빠른 전환**
> **1차: 카카오톡 챗봇 메뉴/버튼 응답**
>
> **2차: LLM 보조 → 고객 자연어 질문 이해 및 메뉴/시나리오 매핑**
>
> **3차: 해결 불가 시 → 상담사 연결 또는 AS 접수**


---
### 5. 다빛솔루션 챗봇의 적용 시나리오

- **반복/단순 질문**
    - **버튼/메뉴 형태(제품문의, 구매문의, 설정안내)로 신속히 응답**
    - 고객이 바로 원하는 자료(매뉴얼, 다운로드 링크 등)로 안내
- **자연어 기반 질문은 LLM 해석 후 경로 연결**
    - 고객이 자유롭게 입력한 질문을 **LLM이 의도(Intent) 인식**
    - 예: 4단 10열 구성에 어떤 컨트롤러를 쓰면 되나요? 
        - LLM이 `제품 추천/구성 문의`로 분류
        - 챗봇이 컨트롤러별 최대 표출 가능 모듈 페이지 연결 버튼 제시
    - 예: 최신 프로그램이 없어 어디서 찾아?
        - LLM이 `프로그램/다운로드`로 분류 
        - 챗봇이 다운로드 버튼 제시해서 다운로드 페이지로 이동
    - 즉, 대화형 응답 대신 **명확한 선택지 제공**으로 빠른 해결 유도
- **복잡하거나 예외적인 상황은 직원 연결 전환**
    - 고객 설명이 모호하거나, 제품별로 다양한 예외 케이스가 포함된 경우
    - 챗봇이 **핵심 정보만 수집** (제품명, 펌웨어 버전, 문제 상황 등) 후 브랜드마케팅팀/기술지원팀으로 연결
    - 예: “프로토콜을 보냈는데 응답이 불규칙하게 와요. 왜 그런 거죠?” → 챗봇이 “사용 중인 모델/펌웨어 버전/발생 빈도”를 묻고, 직원에게 전달
