# Git 병합 규칙

> 이 저장소의 Git 병합 규칙

## 병합 설정

이 저장소에서는 **모든 병합에서 분기점이 명확하게 보이도록** 설정되어 있습니다.

### 설정 내용

```bash
merge.ff = false
```

### 설정 위치

`.git/config` 파일의 `[merge]` 섹션에 저장됩니다.

### 효과

- ✅ 모든 병합에서 병합 커밋이 생성됩니다
- ✅ Git Graph에서 브랜치 분기점이 명확하게 보입니다
- ✅ 브랜치 작업 이력이 보존됩니다

### 확인 방법

```bash
# 설정 확인
git config --get merge.ff

# 출력: false
```

### 병합 예시

```bash
# 이제 --no-ff 옵션 없이도 병합 커밋이 생성됩니다
git merge 브랜치명
```

**결과 그래프:**
```
*   abc1234 (main) Merge branch '브랜치명'
|\
| * def5678 (브랜치명) 작업 내용
* e3f8e15
```

---

## 설정 변경 방법

### 로컬 저장소에만 적용 (현재 설정)

```bash
git config merge.ff false
```

### 전역 설정 (모든 저장소에 적용)

```bash
git config --global merge.ff false
```

### 설정 제거 (Fast-forward 허용)

```bash
git config --unset merge.ff
```

---

## 참고

- 이 설정은 `.git/config` 파일에 저장됩니다
- 팀원들은 각자 설정해야 합니다 (저장소에 자동으로 공유되지 않음)
- 또는 전역 설정으로 모든 저장소에 적용할 수 있습니다
